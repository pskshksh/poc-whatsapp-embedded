1 Prerequisites & Setup
Required Status: Solution Partner OR Tech Provider (completed Meta onboarding)
Facebook Login for Business Configuration

Client OAuth Settings (All must be "Yes"):

  • Client OAuth login: Yes
  • Web OAuth login: Yes
  • Enforce HTTPS: Yes
  • Embedded Browser OAuth Login: Yes
  • Login with JavaScript SDK: Yes
Create Configuration

Configuration Details:

  • Login variation: "WhatsApp Embedded Signup"
  • Permissions: whatsapp_business_management, whatsapp_business_messaging
  • Configuration ID: Generated and saved for implementation

3 User Authorization
User ‚Üí Client App

User clicks "Connect WhatsApp Business" button

Client App ‚Üí Facebook SDK

Launch Embedded Signup

const launchWhatsAppSignup = () => { FB.login(fbLoginCallback, { config_id: 'YOUR_CONFIGURATION_ID', response_type: 'code', override_default_response_type: true, extras: { setup: {}, featureType: '', // '', 'only_waba_sharing', 'whatsapp_business_app_onboarding' sessionInfoVersion: '3' } }); }

Feature Types:

  • '' (blank): Default flow
  • only_waba_sharing: Bypass phone number screens
  • whatsapp_business_app_onboarding: WhatsApp Business App onboarding
  • marketing_messages_lite: Marketing Messages Lite API

6 Server-Side Processing
Solution Partner Flow
Step 1: Exchange Token
GET /v23.0/oauth/access_token
  • client_id: APP_ID
  • client_secret: APP_SECRET
  • code: AUTHORIZATION_CODE

Returns:

BUSINESS_INTEGRATION_SYSTEM_USER_TOKEN
Step 2: Subscribe Webhooks
POST /v23.0/{WABA_ID}/subscribed_apps
  • Authorization: Bearer BUSINESS_TOKEN
Step 3: Share Credit Line
POST /v23.0/{EXTENDED_CREDIT_LINE_ID}/whatsapp_credit_sharing_and_attach
  • waba_currency: USD/EUR/GBP/etc
  • waba_id: CUSTOMER_WABA_ID
  • Authorization: Bearer SYSTEM_TOKEN
Step 4: Register Phone Number
POST /v23.0/{BUSINESS_PHONE_NUMBER_ID}/register
{ "messaging_product": "whatsapp", "pin": "6-digit-PIN" }
Step 5: Test Message (Optional)
POST /v23.0/{BUSINESS_PHONE_NUMBER_ID}/messages
{ "messaging_product": "whatsapp", "recipient_type": "individual", "to": "WHATSAPP_USER_NUMBER", "type": "text", "text": {"body": "Test message"} }

7 Error Handling & Edge Cases

Abandoned Flow

{ data: { current_step: 'SCREEN_NAME' }, type: 'WA_EMBEDDED_SIGNUP', event: 'CANCEL' }

Screen Names:

  • BUSINESS_ACCOUNT_SELECTION
  • WABA_PHONE_PROFILE_PICKER
  • WHATSAPP_BUSINESS_PROFILE_SETUP
  • PHONE_NUMBER_SETUP
  • PHONE_NUMBER_VERIFICATION
  • PERMISSIONS

User Reported Errors

{ data: { error_message: 'DESCRIPTION', error_id: 'ERROR_ID', session_id: 'SESSION_ID', timestamp: 'UNIX_TIMESTAMP' }, type: 'WA_EMBEDDED_SIGNUP', event: 'CANCEL' }

Token Exchange Errors

  • Invalid authorization code
  • Code expired (30-second TTL)
  • Invalid app credentials
  • Network/API timeouts

Business Account Errors

  • Insufficient permissions
  • Business verification required
  • Account creation limits reached
  • Policy violations
